# Module 1: GenAI and the Developer Landscape

## Learning objectives

- Explain the current LLM landscape and how to choose the right model/tool (ChatGPT, Copilot, Codex, Claude).
- Articulate public vs enterprise AI: data handling, confidentiality, what goes where.
- Describe capabilities and limitations: hallucinations, cut-offs, bias, false confidence.
- Address scepticism honestly: AI changes what good work looks like, not whether good work is needed.
- Recognise when to trust output and when to verify.

## Audience notes

- Room is **already tool-aware** — do not re-introduce AI from scratch.
- Mixed Copilot adoption: calibrate with a brief show-of-hands; experienced users can contribute examples.
- **Three BAs** in the room: content is accessible throughout; they need vocabulary for model choice and verification, not coding depth.
- Some sceptics ("cheating", job threat) — acknowledge concerns; reframe toward deliberate, verifiable use.

## Suggested talking points

- What generative AI is — plain language, no hype
- LLM landscape: what Copilot, Codex, and Claude are each optimised for
- Public vs enterprise AI; VS Code integrations in a regulated context
- Capabilities & limitations; trust but verify
- When to rely on AI vs double-check or avoid
