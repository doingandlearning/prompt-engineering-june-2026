# Refinement examples — iterative prompt debugging

## Round 1: Initial prompt (produces poor output)

**Prompt:**
Generate an ASP.NET Core controller for User

**Typical poor output (use as backup if live run fails):**
- Uses service locator / manual DI instead of constructor injection
- .NET 5 / older style (e.g. Newtonsoft attributes, no minimal API awareness)
- No error handling; may return raw null or no status code
- Single endpoint, no clarity on status codes or empty-list behaviour

**Diagnosis:**
- What's missing? — Version (.NET 8?), error handling, response type
- What's wrong? — No constructor injection, old patterns
- What would make it better? — Add context (.NET 8, C# 12) and constraints (constructor injection, IActionResult, error handling)

---

## Round 1: Refined prompt (after first iteration)

**Prompt:**
Generate an ASP.NET Core 8 REST controller using C# 12. Use constructor injection, return IActionResult wrapping a List<User>, and include proper error handling. Don't return null — return an appropriate HTTP status code instead.

**What we added:**
- Context: ASP.NET Core 8, C# 12
- Constraint: Constructor injection
- Constraint: IActionResult wrapping List<User>, proper error handling
- Constraint: Don't return null — use status codes

---

## Round 2 (optional): Second iteration — fix one more thing

**Problem:** The generated controller returns 404 when the user list is empty. We want 200 with an empty array.

**Refined prompt (add to previous):**
For an empty list, return 200 OK with an empty array in the body, not 404.

**Lesson:** Iterative refinement often takes 2–3 rounds. Read the output, pick one concrete fix, add it to the prompt, re-run.
